Visualisation

The simple graph has brought more information to the data analyst’s mind than any other device. — John Tukey

  • Data visualization is the creation and study of the visual representation of data.
  • There are many tools for visualizing data (R is one of them), and many approaches/systems within R for making data visualizations (ggplot2 is one of them, and that is what we are going to use).

The ggplot2 package

Introduction

Powerful and complex data visualisations can be achieved in a systematic fashion using the ggplot2 package, which implements a grammar of graphics that enables us to concisely describe the components of a graphic.

Notes:

  • Contrary to what we prescribed, we will deliberately import all functions from ggplot2 into our R session so that we do not have to repeatedly write ggplot2:: in front of each function from ggplot2 that we call. This is because, when it comes to plotting, an enormous quantity of ggplot2 functions are used so the code would become unnecessarily complex if we decided not to import ggplot2;
  • Similarly, we will illsutrate the features of ggplot2 using the dplyr::starwars, so we will extract it from dplyr to be able to access it by simply calling starwars.

Both of these tasks are performed by the following lines of code:

library(ggplot2)
starwars <- dplyr::starwars

Let us start by looking at this grammar syntax from an example. We first plot the mass of each character against their height:

ggplot(data = starwars, mapping = aes(x = height, y = mass)) +
  geom_point()
Warning: Removed 28 rows containing missing values (geom_point).

What does the following modification achieve?

ggplot(data = starwars, mapping = aes(x = height, y = mass)) +
  geom_point() +
  geom_smooth()

Note: When constructing a plot, you do not apply consectutive processes to an original data set. Rather, you want to add layers for plotting different data representations. The operator to add layers is therefore the natural + addition operator and not the %>% pipe operator.

Specifying labels

You can give your plot a proper title and subtitle, specify axis labels and rename the variables mapped to aesthetics with more human-readable names. This is the job of the labs() function:

ggplot(data = starwars, mapping = aes(x = height, y = mass)) +
  geom_point() +
  labs(

    x = "Height (cm)", 
    y = "Weight (kg)"
  )

Aesthetics

Visual characteristics of plotting characters that can be mapped to a specific variable in the data are:

  • label
  • shape
  • colour
  • fill
  • size
  • alpha (transparency)

There are two types of variables:

  • Continuous variable are measured on a continuous scale
  • Discrete variables are measured (or often counted) on a discrete scale

Mapping continuous or discrete variables to the previous aesthetics will behave differently:

aesthetics discrete continuous
label value for each (converted to text) shouldn’t be used
shape different shape for each shouldn’t (and doesn’t) work
color rainbow of colors gradient
fill rainbow of colors gradient
size discrete steps linear mapping between radius and value
alpha shoudn’t be used linear mapping between radius and value

You might also want to set these features to fixed values, not depending on a variable:

  • Use aesthetics for mapping features of a plot to a variable;
  • Define the features directly in the geom_*() function for customization not mapped to a variable.

Faceting

  • Smaller plots that display different subsets of the data
  • Useful for exploring conditional relationships and large data

You have two functions in ggplot2 to do that:

  • facet_grid(): Lay out panels in a grid;
  • facet_wrap(): Wrap a 1d ribbon of panels into 2d.
ggplot(data = starwars, mapping = aes(x = height, y = mass)) +
  facet_grid(cols = vars(gender)) +
  geom_point() +
  labs(
    title = "Mass vs. height of Starwars characters", 
    subtitle = "Faceted by gender"
  ) + 
  theme(aspect.ratio = 1)
Warning: Removed 28 rows containing missing values (geom_point).

Important: Take a close look at the syntax to specify the facetting variables. They need to be specified within a call to the function vars(). This is how ggplot2 knows that it should find the variables in the data set associated with the current plot.

Visualization in higher dimensions

ggplot2 extensions

The ggplot2 package is now widely used worldwide. As a result, a number of people have been developing extensions to perform all kinds of plots. It is highly probable that, if you do not manage to draw a plot you have on your mind easily with your knowledge of ggplot2, then someone probably has published an extension already that perfectly fits your needs. You can consult at any time the up-to-date list of ggplot2 extensions.

Heatmaps

You can visualize a third dimension by color-coding a 2D plane. This is what does a heatmap. There are several ways to draw heatmaps in R. We recommend the d3heatmap package which provides interactive heatmaps.

Data visualization of 3D data in 3D

Lots of packages offer 3D visualization tools. We recommend the plot3D package, which is particularly well implemented and easy to use. A nice introductory tutorial can be found here.

Visualizing nD data

Visualization of nD data seems impossible on paper. One way to actually do this is to visualize projections of the nD data into \(\mathbb{R}\), \(\mathbb{R}^2\) or \(\mathbb{R}^3\). You then create a tour of your data by generating a smooth sequence of projections and display your data as a movie. In R, this is easily achieved by the tourr package.

To demonstrate this technique we will use a data set in the tourr package called flea, which contain measurements of flea beetle characteristics for 3 different species:
